# Incident Response Workflow

Goal:
Restore system functionality and identify root cause.

## Step 1 — Assess Incident

Use skill:

- incident_responder

Determine:

- affected systems
- impact
- recent changes

---

## Step 2 — Stabilize System

Actions may include:

- restart services
- rollback deployment
- isolate failing components

---

## Step 3 — Root Cause Analysis

Identify:

- failure point
- underlying cause

---

## Step 4 — Recovery

Implement:

- configuration fixes
- code patches
- infrastructure changes

---

## Step 5 — Prevention

Recommend:

- monitoring improvements
- alerts
- architectural fixes
