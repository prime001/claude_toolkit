# Glossary

This file defines project-specific terms, acronyms, and concepts that Claude should interpret correctly.

Add terms here as the repo grows.

## Example Entries

### API
Application Programming Interface.

### CI
Continuous Integration.

### CD
Continuous Delivery / Deployment.

### Worker
A background process responsible for async or scheduled work.

### Service
A deployable backend component or internal logical unit.

### Repo
This code repository.

### Environment
A target runtime such as local, dev, test, staging, or production.

## Guidance for Claude
- prefer glossary meanings over generic assumptions when terms are ambiguous
- if the same term has both a general and repo-specific meaning, use the repo-specific meaning
- keep this file updated with internal acronyms and domain language
